# ทำไมค้นหาด้วยภาพแล้วได้สินค้าอื่น / รูปไม่ตรงกับชื่อ

## สาเหตุหลัก

1. **รูปของสินค้าในระบบผิด**  
   ถ้าในตาราง `image` สินค้าชิ้นนั้น (เช่น Sony WH-1000XM5) ถูกผูกกับ URL รูปอื่น (เช่น รูปชายหาด) ทั้งแอปและ AI จะใช้รูปนั้น  
   → เวลาค้นหาด้วยภาพที่คล้ายชายหาด ระบบจะคืนสินค้านั้น และหน้ารายการก็แสดงรูปชายหาด → ดูเหมือน “ได้สินค้าอื่น”.

2. **Visual Search คืนจาก “รูปที่คล้ายที่สุด”**  
   AI ไม่ได้ดูจากชื่อสินค้า แค่เปรียบเทียบรูปที่คุณส่งกับรูปสินค้าใน DB แล้วคืน product ที่รูปคล้ายที่สุด  
   → ถ้ารูปใน DB ผิด ผลที่ได้จะดูไม่ตรงกับชื่อ.

## วิธีตรวจสอบใน Database

รันใน MySQL (เปลี่ยน `ชื่อสินค้า` เป็นชื่อที่เห็นในแอป):

```sql
-- ดูว่าสินค้านั้นมีรูปอะไรบ้าง (เรียงตาม id รูป)
SELECT p.id AS product_id, p.title, i.id AS image_id, i.url
FROM product p
JOIN image i ON i.productId = p.id
WHERE p.title LIKE '%Sony WH-1000XM5%'
ORDER BY p.id, i.id;
```

- ถ้า `url` ของรูปแรกเป็นรูปที่ไม่ตรงกับชื่อสินค้า (เช่น รูปชายหาด) = **ข้อมูลผิด** ต้องแก้ที่รูปของสินค้านั้น (อัปโหลดรูปที่ถูกหรือลบ/แก้ record ใน `image`).

## ทางแก้ (สำหรับแอดมิน/ร้านค้า)

- เปิดหน้ารายละเอียดสินค้าในระบบหลังบ้าน (หรือแอปแอดมิน) แล้ว **เปลี่ยนรูปหลักของสินค้า** ให้เป็นรูปที่ตรงกับชื่อ (หรือลบรูปผิดออก).
- หลังจากแก้รูปใน DB แล้ว ให้ **reindex AI Service** เพื่อให้การค้นหาด้วยภาพใช้รูปใหม่: เรียก `GET http://localhost:8000/reindex` (หรือตาม URL ของ AI Service ที่ใช้).

## สรุปให้ผู้ใช้เห็นในแอป (ถ้าต้องการ)

สามารถเพิ่มข้อความใต้ผลการค้นหาด้วยภาพ เช่น:

- “ผลการค้นหาจากความคล้ายของรูปภาพ อาจไม่ตรงกับชื่อสินค้า 100%”
- “ถ้ารูปไม่ตรงกับชื่อสินค้า แจ้งร้านค้าเพื่อแก้ไขรูปสินค้าได้”
