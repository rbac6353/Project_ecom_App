## 🆕 การปรับปรุงล่าสุด (รอบแก้ไขนี้)

- **Auto-Cancellation System (ระบบยกเลิกออเดอร์อัตโนมัติ)**
  - เพิ่มฟิลด์ `confirmationDeadline` และ `isAutoCancelled` ใน `Order` entity
  - สร้าง Cron Job (`@Cron(CronExpression.EVERY_MINUTE)`) ใน `order-cleanup.task.ts` เพื่อตรวจสอบและยกเลิกออเดอร์ที่หมดเวลา
  - Logic: เมื่อออเดอร์เปลี่ยนสถานะเป็น `PENDING_CONFIRMATION` จะตั้งค่า `confirmationDeadline = createdAt + 24 ชั่วโมง`
  - เมื่อหมดเวลา Cron Job จะ:
    - เปลี่ยนสถานะเป็น `CANCELLED`
    - ตั้งค่า `isAutoCancelled = true`
    - คืนสต็อกสินค้าอัตโนมัติ (ใช้ Database Transaction + Pessimistic Locking)
    - ส่งการแจ้งเตือน
  - Frontend: เพิ่ม Countdown Timer ใน `OrderCard.tsx` แสดงเวลาที่เหลือ (`HH:mm:ss`) และ Auto-refresh เมื่อหมดเวลา
  - SQL Migration: `add_order_confirmation_deadline.sql`

- **Flash Sale Platform (Scarcity Management แบบ Shopee/Lazada)**
  - ออกแบบ **Flash Sale Campaign** แยกจากตารางสินค้า โดยใช้ตาราง `flash_sale` (Campaign/Time Slot) และ `flash_sale_item` (สินค้าในแคมเปญ พร้อมราคา `discountPrice`, โควตา `limitStock`, จำนวนขาย `sold`, จำกัดต่อคน `maxPerUser`).
  - สร้าง `FlashSaleModule` (Backend) พร้อม Service/Controller ครบชุด:
    - Public: `GET /flash-sales/current` เพื่อดึงแคมเปญที่กำลังรันหรือรอบถัดไป (พร้อมรายการสินค้าและquota).
    - Admin: `GET /flash-sales/admin/all`, `POST /flash-sales/admin/create`, `POST /flash-sales/admin/:id/items`, `PATCH /flash-sales/admin/:id`, `DELETE /flash-sales/admin/:id`.
  - รวม Flash Sale เข้ากับ `OrdersService`:
    - ตรวจสอบว่า product อยู่ใน Flash Sale, เวลาอยู่ในช่วงแคมเปญ, quota ยังไม่หมด และ (เตรียมรองรับ) จำกัดจำนวนต่อผู้ใช้ (`maxPerUser`).
    - ใช้ `reserveFlashSaleStock()` ที่ทำงานแบบ atomic และใช้เงื่อนไข `sold < limitStock` เพื่อป้องกัน oversell.
    - ถ้าเข้าเงื่อนไข Flash Sale → ใช้ `discountPrice` แทนราคาปกติ, ถ้าไม่เข้า → กลับไปใช้ราคาปกติ.
  - แก้ไข schema และ entity ให้ตรงกับฐานข้อมูลจริง (เช่น `specialPrice` → `discountPrice`, `stock` → `limitStock`, เพิ่ม `name`, `description`, `createdAt` ใน `flash_sale` และ `createdAt` ใน `flash_sale_item`) พร้อม SQL scripts (`create_flash_sale_tables.sql`, `add_flash_sale_columns.sql`, `quick_fix_flash_sale.sql`).

## 📦 Backend Structure - รายละเอียด API Endpoints

### 1. **AuthModule** (`/auth`)

#### Endpoints:
- `POST /auth/login` - เข้าสู่ระบบ (Email/Password)
  - Request: `{ email: string, password: string }`
  - Response: `{ access_token: string, user: User }`
- `POST /auth/register` - สมัครสมาชิก
  - Request: `{ email: string, password: string, name: string }`
  - Response: `{ access_token: string, user: User }`
- `GET /auth/profile` - ดูข้อมูลโปรไฟล์ (JWT Required)
  - Headers: `Authorization: Bearer <token>`
  - Response: `User`

#### Features:
- ✅ Password hashing ด้วย bcrypt
- ✅ JWT Token generation
- ✅ JWT Strategy (Passport)
- ✅ JWT Guard สำหรับ Protected Routes
- ✅ Email validation (ตรวจสอบ email ซ้ำ)

### 2. **ProductsModule** (`/products`)

#### Endpoints:
- `GET /products` - ดึงสินค้าทั้งหมด
  - Query: `?categoryId=1&page=1&limit=10&keyword=...&minPrice=...&maxPrice=...&sortBy=...` (optional)
  - Response: `{ data: Product[], total: number, page: number, last_page: number }`
  - Features: Pagination, Filtering, Sorting, Validation (ป้องกันค่าติดลบ, สลับค่าเมื่อ min > max)
- `GET /products/search?keyword=...` - ค้นหาสินค้า
  - Query: `keyword` (required)
  - Response: `Product[]`
- `GET /products/suggestions?keyword=...` - คำแนะนำการค้นหา (Autocomplete)
  - Query: `keyword` (required)
  - Response: `string[]` (กรองเฉพาะคำที่ขึ้นต้นด้วย keyword)
  - Features: กรองคำที่ขึ้นต้นด้วยตัวอักษรที่พิมพ์ (เช่น พิมพ์ "ก" -> แสดง "กางเกง", "กระเป๋า")
- `GET /products/:id` - ดึงรายละเอียดสินค้า
  - Params: `id` (number) - Validation: ต้องเป็นตัวเลขบวก
  - Response: `Product` (with relations: images, category, store, reviews)
- `POST /products/visual-search` - ค้นหาด้วยภาพ (Visual Search)
  - Content-Type: `multipart/form-data`
  - Body: `image` (file)
  - Response: `Product[]` (เชื่อมต่อกับ AI Service)
- `POST /products` - สร้างสินค้าใหม่ (JWT Required, Seller/Admin)
  - Content-Type: `multipart/form-data`
  - Body: `name`, `description`, `price`, `stock`, `categoryId`, `image` (file)
  - Response: `Product`

#### Features:
- ✅ Product CRUD Operations
- ✅ Category Filtering
- ✅ Text Search (keyword-based)
- ✅ Search Suggestions/Autocomplete (กรองเฉพาะคำที่ขึ้นต้นด้วย keyword)
- ✅ Visual Search (เชื่อมต่อกับ AI Service)
- ✅ Product Relations (Images, Category, Store, Reviews)
- ✅ File Upload (Product Images)
- ✅ Static File Serving
- ✅ Pagination (page, limit, total, last_page)
- ✅ Price Range Filtering (minPrice, maxPrice) with validation
- ✅ Sorting (price_asc, price_desc, sold, related)
- ✅ Input Validation (ป้องกันค่าติดลบ, invalid ID, out of range)

### 5. **OrdersModule** (`/orders`) - JWT Required

#### Endpoints:
- `POST /orders` - สร้างออเดอร์
  - Headers: `Authorization: Bearer <token>`
  - Body: `{ shippingAddress: string, shippingPhone: string, couponCode?: string }`
  - Response: `Order` (with products)
- `GET /orders` - ดูออเดอร์ทั้งหมดของ User
  - Headers: `Authorization: Bearer <token>`
  - Response: `Order[]`
- `GET /orders/admin/all` - ดูออเดอร์ทั้งหมด (Admin/Seller)
  - Headers: `Authorization: Bearer <token>`
  - Response: `Order[]` (with user info)
- `GET /orders/:id` - ดูรายละเอียดออเดอร์
  - Headers: `Authorization: Bearer <token>`
  - Params: `id` (number)
  - Response: `Order` (with products)
- `PATCH /orders/:id/status` - อัปเดตสถานะออเดอร์ (Admin/Seller)
  - Headers: `Authorization: Bearer <token>`
  - Body: `{ status: string, trackingNumber?: string, logisticsProvider?: string }`
  - Response: `Order`
- `POST /orders/:id/refund` - ขอคืนเงิน (Customer)
  - Headers: `Authorization: Bearer <token>`
  - Body: `{ reason: string }`
  - Response: `Order`
  - **เงื่อนไข**: ออเดอร์ต้องมีสถานะ `COMPLETED` และ `refundStatus` เป็น `NONE`
- `PATCH /orders/:id/refund/decide` - อนุมัติ/ปฏิเสธการคืนเงิน (Seller/Admin)
  - Headers: `Authorization: Bearer <token>`
  - Body: `{ decision: 'APPROVED' | 'REJECTED' }`
  - Response: `Order`
  - **Features**: เมื่ออนุมัติจะคืนสต็อกสินค้าอัตโนมัติ
- `GET /orders/:id/invoice` - ดาวน์โหลดใบเสร็จรับเงิน (PDF)
  - Headers: `Authorization: Bearer <token>`
  - Response: PDF File

#### Order Status Flow:
- **Normal Flow**: `PENDING` → `VERIFYING` → `PENDING_CONFIRMATION` → `PROCESSING` → `READY_FOR_PICKUP` → `RIDER_ASSIGNED` → `PICKED_UP` → `SHIPPED` → `OUT_FOR_DELIVERY` → `DELIVERED` → `COMPLETED`
- **Cancellation**: `CANCELLED` - ออเดอร์ถูกยกเลิก (คืนสต็อกอัตโนมัติ)
  - **Auto-Cancellation**: เมื่อออเดอร์อยู่ในสถานะ `PENDING_CONFIRMATION` และ `confirmationDeadline` หมดอายุ (24 ชั่วโมง) ระบบจะยกเลิกอัตโนมัติผ่าน Cron Job (รันทุก 1 นาที)
  - **Frontend**: แสดง Countdown Timer (`HH:mm:ss`) ใน `OrderCard.tsx` และ Auto-refresh เมื่อหมดเวลา
- **Refund Flow**: `COMPLETED` → `REFUND_REQUESTED` (เมื่อลูกค้าขอคืน) → `REFUNDED` (เมื่ออนุมัติ)
- **Payment Expiry**: Auto-cancel เมื่อ `paymentExpiredAt` หมดอายุ (คืนสต็อกอัตโนมัติ)

#### Refund Status (RefundStatus Enum):
 - `NONE` - ไม่มีการขอคืนเงิน (ค่าเริ่มต้น)
 - `PENDING` - อยู่ระหว่างรอการดำเนินการ/ตรวจสอบการคืนเงิน
 - `REQUESTED` - ลูกค้าขอคืนเงิน (ต้องเป็นออเดอร์ `COMPLETED` เท่านั้น)
 - `APPROVED` - ร้านอนุมัติการคืนเงิน (คืนสต็อกสินค้าและ variant อัตโนมัติ)
 - `REJECTED` - ร้านปฏิเสธการคืนเงิน

**Business Rules**:
- ✅ ขอคืนเงินได้เฉพาะออเดอร์ที่ `COMPLETED` เท่านั้น
- ✅ ไม่สามารถขอคืนซ้ำได้ (ต้องเป็น `refundStatus = NONE`)
- ✅ เมื่ออนุมัติการคืนเงิน จะคืนสต็อกสินค้าและ variant อัตโนมัติ

## 🤖 AI Visual Search Service

### Python FastAPI Microservice (`AI_Service/`)

#### Features:
- ✅ FastAPI Framework
- ✅ MobileNetV3 Model (Feature Extraction)
- ✅ Cosine Similarity (Product Matching)
- ✅ MySQL Integration (เชื่อมต่อกับ Backend Database)
- ✅ Image Indexing (In-memory Feature Vectors)
- ✅ REST API Endpoint: `POST /visual-search`

#### Architecture:
- **Port**: 8000
- **Database**: เชื่อมต่อกับ `ecom1` database
- **Model**: MobileNetV3 (Pre-trained)
- **Similarity Threshold**: 0.3 (ปรับได้)

#### Integration:
- Backend (`POST /products/visual-search`) ส่ง request ไปที่ AI Service
- AI Service คืนค่า Product IDs ที่คล้ายกัน
- Backend ดึงข้อมูลสินค้าจาก Database และส่งกลับ Frontend

## 🐛 Known Issues & Solutions

1. **Visual Search 404 Error**: 
   - **Solution**: ใช้ `npm run start:dev` แทน `npm start`

2. **Navigation Error**: 
   - **Solution**: แก้ไขโดยลบ comments และ whitespace ใน Navigator files

3. **AsyncStorage Token**: 
   - **Status**: แก้ไขแล้วใน AuthContext

4. **API URL Configuration**:
   - **Solution**: ตรวจสอบ IP Address ของเครื่องและอัปเดตใน `api/client.ts`

5. **Worklets Error (react-native-reanimated)**:
   - **Solution**: ใช้ React Native Animated API แทน reanimated สำหรับ Expo Go
   - **Status**: แก้ไขแล้ว (ใช้ Animated API สำหรับ Skeleton และ Heart Animation)

6. **Wishlist 500 Error**:
   - **Cause**: `req.user.userId` แทน `req.user.id`
   - **Solution**: แก้ไขใน WishlistController
   - **Status**: แก้ไขแล้ว

7. **Refund Request Error - "ต้องเป็นออเดอร์ที่สำเร็จแล้วจึงจะขอคืนได้"**:
   - **Cause**: ลูกค้าพยายามขอคืนเงินจากออเดอร์ที่ยังไม่ `COMPLETED`
   - **Solution**: ระบบจะตรวจสอบว่า `orderStatus === COMPLETED` และ `refundStatus === NONE` ก่อนอนุญาตให้ขอคืน
   - **Status**: ทำงานตามที่ออกแบบ (Business Rule)
   - **Note**: ออเดอร์ต้องผ่านขั้นตอนทั้งหมดจนถึง `COMPLETED` (ลูกค้ายืนยันรับสินค้า) ก่อนจึงจะขอคืนได้
